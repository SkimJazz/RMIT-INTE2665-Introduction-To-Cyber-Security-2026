# 5. What are the important host and network configuration files?


* `/etc/hosts`
* `/etc/hostname` 
* `/etc/resolv.conf` 

What information is contained in each configuration file?

Info in these files are Core host + Network configuration files.

```bash
cat /etc/hosts
```
What does it do? => Local hostname-to-IP address mappings
```zsh
┌──(kali㉿kali)-[~]
└─$ cat /etc/hosts      
127.0.0.1       localhost
127.0.1.1       kali
::1             localhost ip6-localhost ip6-loopback
ff02::1         ip6-allnodes
ff02::2         ip6-allrouters

```
**Why these files matter:**

- Checked before DNS
- Can override real domains
- Used in phishing, malware redirection, and testing

**🔐 Security relevance:**

- Host file poisoning
- Bypassing DNS-based controls


<br>

```bash
cat /etc/hostname
```
What does it do? => Defines the system's hostname
```zsh
┌──(kali㉿kali)-[~]
└─$ cat /etc/hostname
kali

```
**Why theses files matter:**

- Used in logs
- Used in network identification
- Helps identify systems in incident response

**🔐 Security relevance:**

- Asset identification
- Host impersonation in poorly monitored networks


<br>

```bash
cat /etc/resolv.conf
```
What does it do? => DNS resolver configuration
```zsh
┌──(kali㉿kali)-[~]
└─$ cat /etc/resolv.conf          
# Generated by NetworkManager
search home
nameserver xxx.xxx.xx.x
nameserver xxxx:xxxx:xxxx:x::x
```
**Why these files matter:**

- Controls which DNS server is trusted
- Determines how domain names are resolved

**🔐 Security relevance:**

- DNS hijacking
- Man-in-the-middle attacks
- Command-and-control redirection

>**⚠️ Note (important but not examinable yet): 
> On modern systems this file may be auto-managed by NetworkManager or systemd-resolved**



**big picture discussion:**

These files define:

- Who this machine thinks it is
- How it resolves names
- Where network traffic goes
- Which systems it trusts first

From a security perspective, these files are:
- High-value targets
- Common persistence mechanisms
- Frequently checked during incident response

**Discussion:**

_Important host and network configuration files in Linux include `/etc/hosts`, which maps hostnames to IP addresses locally; `/etc/hostname`, which defines the system’s name; and `/etc/resolv.conf`, which specifies DNS servers used for name resolution. The `/etc/services` file lists well-known services and their associated port numbers. These files are important because they control how the system identifies itself and communicates on the network._
